1. ¿Qué diferencia existe entre paralelismo y concurrencia?
La concurrencia organiza varias tareas al mismo tiempo; el paralelismo las ejecuta realmente al mismo tiempo.

2. ¿Qué problema ocurre cuando varios hilos acceden al mismo recurso?
Pueden generarse errores o inconsistencias en los datos compartidos.

3. ¿Qué es una condición de carrera (Race Condition)?
Es un problema que ocurre cuando varios hilos modifican un mismo recurso sin control y el resultado depende del orden de ejecución.

4. ¿Por qué es importante sincronizar el acceso a recursos compartidos?
Porque evita errores, protege los datos y asegura que un solo hilo modifique el recurso a la vez.
